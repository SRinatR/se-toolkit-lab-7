This bot will be built in small steps so that every task stays testable and easy to debug. For Task 1, the goal is to create a clean scaffold with an entry point, plain handler functions, config loading, and an offline CLI test mode. The most important design decision is to keep handlers independent from Telegram. A handler should accept simple input values and return plain text. This allows the same logic to be reused both in local test mode and later in the Telegram transport layer.

For Task 2, I will connect the handlers to the LMS backend through a small API client service. Commands like /health, /labs, and /scores will call backend endpoints and format readable responses. For Task 3, I will add natural language routing through an LLM client and tool calls, while preserving the same handler and service boundaries. For Task 4, I will containerize the bot, integrate it into docker-compose, and document deployment on the VM. This structure should keep the code simple, testable, and easy to extend.
